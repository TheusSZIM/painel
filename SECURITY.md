## Reportando Vulnerabilidades

Levamos a segurança muito a sério. Se você descobrir uma vulnerabilidade de segurança, por favor siga estes passos:

### Processo de Reporte

1. **Não divulgue publicamente** - Não crie uma issue pública
2. **Envie um email** para [seu-email@exemplo.com] com:
   - Descrição detalhada da vulnerabilidade
   - Passos para reproduzir
   - Possível impacto
   - Sugestões de correção (se houver)

### O que Esperar

- **Confirmação:** Responderemos em até 48 horas
- **Atualizações:** Você receberá atualizações a cada 7 dias
- **Resolução:** Trabalharemos para resolver em até 30 dias
- **Crédito:** Você será creditado (se desejar) após a correção

### Escopo

Estamos interessados em vulnerabilidades relacionadas a:

- Injeção de código (XSS)
- Vazamento de dados sensíveis
- CSRF
- Problemas de autenticação/autorização
- Configurações inseguras

### Fora do Escopo

Não consideramos vulnerabilidades:

- Ataques de força bruta em autenticação
- Problemas de rate limiting
- Vulnerabilidades em dependências de terceiros (reporte aos mantenedores)
- Problemas que requerem acesso físico ao dispositivo

## Boas Práticas de Segurança

### Para Usuários

- Mantenha seu navegador atualizado
- Use senhas fortes e únicas
- Desconfie de links suspeitos
- Reporte comportamentos anômalos

### Para Desenvolvedores

- Nunca comite senhas ou chaves de API
- Use HTTPS em produção
- Valide todas as entradas de usuário
- Mantenha dependências atualizadas
- Faça code review antes de merge
